Joule Trading Platform

Joule is Trayport’s flagship desktop application for European energy and commodity trading. It provides traders with advanced market connectivity, order management, and execution capabilities in a high-performance, real-time environment. The application is built using .NET and WPF, with strong integration to Excel for analysis and automation through Excel-DNA.

In this role, I contributed to the modernization of the platform as part of Trayport’s migration to .NET 8, focusing on improving performance, maintainability, and long-term scalability.

I was responsible for:

  • Supporting the migration from .NET 4.8 to .NET 8, with emphasis on networking and localization layers.
  • Modernizing and re-architecting the testing infrastructure, migrating from NUnit Console to dotnet test for improved automation, parallel execution, and CI/CD integration in TeamCity.
  • Optimizing the build system, consolidating project outputs and reducing disk usage by 98%, significantly improving build speed and reliability.
  • Re-engineering the Excel-DNA integration layer, replacing obsolete .NET Remoting with Named Pipes for secure, robust inter-process communication; led a small team of two developers.
  • Leading the AutoTrader Order Entry Dialog revamp, enabling creation of multi-leg spreads with a modernized user interface, new WPF controls, and improved workflow; coordinated delivery between London and Vienna teams.

Technologies used: C#, .NET 8, WPF, Excel-DNA, Named Pipes, NUnit, TeamCity, Git, MVVM
